# Documentation

This project uses the Diataxis documentation model.

Use:

- `docs/tutorials/` for guided learning paths
- `docs/how-to/` for practical task-oriented guides
- `docs/reference/` for factual descriptions of interfaces and contracts
- `docs/explanation/` for rationale, architecture, and trade-offs

Recommended specializations:

- `docs/reference/testing/` for testing contracts and traceability
- `docs/explanation/decisions/` for ADRs

When adding documentation, place it in the correct section instead of mixing
multiple document types in one file.
