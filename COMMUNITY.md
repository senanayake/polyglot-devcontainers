# Community

## Who This Project Is For

Polyglot Devcontainers is for:

- platform and developer-experience teams
- open source maintainers
- engineers who want reproducible local and CI workflows
- teams exploring AI-compatible execution environments
- contributors who value explicit contracts over hidden automation

## How To Participate

You can participate by:

- opening issues
- proposing pull requests
- improving documentation and examples
- contributing templates and scenarios
- testing workflows on real machines and reporting friction

## Contribution Ladder

Common participation paths:

1. Report friction, bugs, or unclear documentation.
2. Improve docs, examples, and presentation.
3. Contribute targeted template, workflow, or scenario changes.
4. Help review, harden, and refine public-facing surfaces.
5. Become a regular contributor with sustained repository knowledge.

## Current Collaboration Model

Today, most collaboration happens through GitHub issues and pull requests.

The project is still in a phase where clarity of scope, presentation, and
working examples matter more than heavy community process.

## Future Direction

As the project matures, community activity may expand to include:

- demos
- walkthroughs
- meetups
- shared scenarios
- contributor onboarding sessions

The goal is not growth for its own sake. The goal is a healthy community around
a clear, useful, reproducible system.
