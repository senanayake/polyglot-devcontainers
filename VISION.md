# Vision

## Purpose

Polyglot Devcontainers exists to make software development more reproducible,
more explicit, and easier to operate across people, machines, and automation.

The project aims to provide an open execution system for modern software
development built from:

- deterministic container environments
- a stable task contract
- reusable starter templates
- published images
- runnable scenarios for supported workflows

## What Polyglot Is

Polyglot is a container-first execution surface for software work.

It helps teams and agents:

- enter known-good environments quickly
- use the same workflow across local work and CI
- keep security and validation in the default path
- package working patterns into reusable starters and scenarios

## What Polyglot Is Not

Polyglot is not:

- a general-purpose reasoning engine
- a hidden automation layer
- a replacement for ecosystem-native tools
- a platform that depends on undocumented behavior

Polyglot should expose execution clearly. Interpretation, planning, and
higher-level reasoning can surround the system, but they should not be hidden
inside the core contract.

## Near-Term Direction

In the near term, the project is focused on:

- improving starter quality and clarity
- strengthening the repository and image contracts
- making examples and scenarios easier to understand and operate
- presenting a clear public identity and contribution surface

## Long-Term Aspiration

Over time, Polyglot should become a trusted open source foundation for
reproducible software development loops:

- predictable for humans
- operable by agents
- useful across languages
- honest about scope and guarantees
