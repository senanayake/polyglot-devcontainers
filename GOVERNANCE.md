# Governance

## Current Model

Polyglot Devcontainers is currently maintained through a maintainer-led model.

That means:

- maintainers set direction
- contributors propose changes through issues and pull requests
- decisions are made in the repository, with visible rationale

## Decision Principles

Maintainers should prefer decisions that improve:

- reproducibility
- clarity
- scope discipline
- explicit behavior
- long-term maintainability

## How Decisions Should Be Made

For meaningful repository changes:

- describe the problem clearly
- keep scope visible
- show the operational impact
- prefer evidence over assertion
- document public-surface changes in the repo

## Contribution Path

Contributors do not need special status to improve the project.

High-quality contributions that align with the project principles are welcome.
Sustained, constructive contribution naturally builds trust and influence.

## Governance Intent

The project should remain open, practical, and understandable.

If governance becomes more formal later, that should happen in response to real
community needs rather than premature process design.
